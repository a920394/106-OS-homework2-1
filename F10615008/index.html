🤔 功能是什麼？

實現的System call

  • mlock(const void *addr，size_tlen)
  • mlockall(int flags)
  • munlock(const void *addr，size_tlen)
  • munlockall(void)

具體功能

mlock

係統調用 mlock 家族允許程序在物理記憶體上鎖住它的部分或全部地址空間。這將阻止Linux 將這個記憶體到交換空間（swap space），即使該程序已有一段時間沒有訪問這段空間。
一個嚴格時間相關的程序可能會希望鎖住物理內存，因為內存頁麵調出調入的時間延遲可能太長或過於不可預知。安全性要求較高的應用程序可能希望防止敏感數據被換出到交換文件中，因為這樣在程序結束後，攻擊者可能從交換文件中恢複出這些數據。
鎖定一個內存區間隻需簡單將指向區間開始的指針及區間長度作為參數調用 mlock。Linux 分配內存到頁(page)且每次隻能鎖定整頁內存，被指定的區間涉及到的每個內存頁都將被鎖定。

munlock

同理,munlock實現對mlock鎖住的物理記憶體的解鎖。

mlockall

mlockall可以同時鎖定當前在使用的所有記憶體，或未來將使用的所有記憶體，或是把全部記憶體鎖定。

munlockall

munlockall將mlockall鎖住的物理記憶體解鎖。

😇 如何實作？

管理記憶體的資料結構——vma

每一個process有一段連續的虛擬記憶體，OS將整個虛擬記憶體劃分成若干個虛擬記憶體區域。使用vma這樣的數據結構來管理虛擬記憶體區域：

Struct vma{
    address vm_end;
    address vm_start;
    bool vm_lockflag;
    ...
    vma *vm_next;
}
  • mlock操作會將相應的vma的vm_lockflag置為1，這個標記會物理內存的回收和交換。
  • munlock操作則將相應的vma的vm_lockflag置為0，取消對相應的vm的鎖定
  • mlockall操作將所有的running的process用到的Memory都進行mlock操作
  • munlcokall對mlockall加鎖的虛擬記憶體進行解鎖。

vma組織方式

採用鏈表和AVL樹來管理vma，鏈表便於我們進行遍歷，AVL樹便於我們進行查找。

使用鏈表維護鎖定頁

使用usable_list、unusable_list、unevictable_list三個鏈表來維護頁

  • usable_list中存放可使用的頁
  • unusable_list中存放已被進程使用了的頁
  • 被鎖定的頁則被我們放入unvictable_list中

解鎖

把加入unvictable_list中的頁放回usable_list中

註

  • 由於內存是按頁分配，也是按頁鎖定，所以如果給出的地址不在page的邊界，會被對齊到下一個page的邊界上。
  • 由於線程共享進程資源，所以線程的vma將繼承vm_lockflag標記，但是fork出的子進程不會繼承vm_flag,調用exec執行其他程序時也不會繼承vm_flag標記。

🐻 預期成效（怎麼用、誰用）

誰用

  • 我們不希望記憶體中的數據被swap時（程式對安全性要求比較高的時候，），可以調用mlock對process使用的記憶體進行鎖定，從而防止敏感數據被用；
  • 一個嚴格時間相關的程序可能會希望鎖住物理內存，因為內存頁麵調出調入的時間延遲可能太長或過於不可預知，使用mlock後可以減少程序的運行時間。

怎么用

System call Description
mlock(const void *addr，size_t len) 將起始地址為addr，大小為len的虛擬記憶體鎖定到物理記憶體
mlockall(int flags) MCL_CURRENT 鎖定當前虛擬記憶體; MCL_FUTURE 鎖定未來虛擬記憶體； MCL_CURRENT|MCL_FUTURE 同時鎖定當前和未來虛擬記憶體
munlock(const void *addr，size_t len) 將起始地址addr，長度為len的物理記憶體解鎖
munlockall(void) 將mlockall()的物理記憶體解鎖
